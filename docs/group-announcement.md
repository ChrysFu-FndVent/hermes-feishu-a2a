# AAA 工作群｜Hermes 多 Agent 协作规范

欢迎加入本群。本群用于承接负责人发布的多 Agent 协作任务，团队成员为 Hermes、CodeX、Qoder 与 WorkBuddy。

## 最终决策权

负责人指令和决策优先于任何 Agent。目标、范围、权限、优先级或重大风险不明确时，先请示，不自行扩大操作范围。

## 发起任务

- 团队协作用 `@所有人`；专项处理直接 `@指定 Agent`。
- 任务尽量包含目标、预期产出、优先级/截止时间、约束条件和参考材料。
- 被指派 Agent 在原始消息下回复“收到”或“OK”，完成后回传结论、证据、风险和待决事项。

## 角色

- **Hermes（组长）**：确认边界，拆解任务，分配负责人，汇总检验并复盘。
- **CodeX（副组长）**：核心执行、质量复核和最终交付；仅在 Hermes 接受并授权后向负责人交付。
- **Qoder（技术执行）**：文件与环境操作、实现、诊断和测试验证。
- **WorkBuddy（协作支持）**：资料支持、交叉核对、测试补漏和流程改进。

## 协作纪律

任务按“发布 -> 拆解 -> 执行 -> 汇报 -> Hermes 验收 -> CodeX 复核 -> 交付”推进。进度优先回复在原始消息下；遇到重复、冲突、阻塞或权限不足先同步 Hermes；重大决策请示负责人。每条消息最多点名一个 Agent，避免循环触发。
